# Aggregat — дизайн системы

> Дата: 2026-07-01 · Статус: утверждён · Модель проектирования: opus

## Цель

Видеть **реально важные вещи** среди потока уведомлений и заниматься ими, а не
тонуть в шуме. Телефон ловит все пуши → сервер группирует, тегирует и оценивает
важность с учётом GTD-структуры жизни → сайт показывает отсортированную ленту
«важного».

## Компоненты

```
Android (NotificationListener → очередь) ──ingest──▶ Server (FastAPI + Postgres) ◀──REST── Сайт (дашборд)
                                                       │ Pipeline: Rules → LLM-router → Grouping
```

1. **Android** — `NotificationListenerService` ловит все уведомления, кладёт в
   локальную очередь (переживает оффлайн), батчами шлёт на сервер.
2. **Server** — FastAPI + Postgres. Принимает сырые пуши, прогоняет через
   пайплайн обогащения, хранит сырые + обогащённые данные, отдаёт REST для сайта.
3. **Сайт** — дашборд: лента, отсортированная по важности, сгруппированная по
   тредам, с фильтрами по зонам/проектам/тегам.

## GTD-ядро (разделы жизни)

- **Area** — постоянная зона ответственности (Работа, Здоровье, Финансы, Семья…).
- **Project** — активный проект с финишем, принадлежит Area.
- **Tag** — произвольная метка.
- Назначаются заводятся пользователем; классификатор привязывает Item к Area/Project.

## Пайплайн классификации

1. **Rules** (бесплатно, детерминированно): по source-app, ключевым словам,
   отправителю → кандидат-теги + базовая важность.
2. Уверенно → готово. Иначе → **LLM-роутер**: `haiku` для рутины, эскалация на
   `opus` для неоднозначного. Возвращает Area/Project, теги, важность 0-100,
   group-key, краткое саммари, предложенное действие.
3. **Grouping** — дедуп + тред похожих Item (тот же app+проект в окне времени).

Провайдер LLM абстрагирован (`LLMProvider`) → можно воткнуть локальный Ollama для
максимальной приватности. **Приватность**: self-host, контент не уходит третьим
лицам; в LLM API уходит минимум данных, либо локальная модель.

## Решения (зафиксировано в брейншторме)

| Вопрос | Решение |
|---|---|
| Разделы жизни | GTD/PARA: Project внутри Area, заводит пользователь |
| Движок тегов | Гибрид: правила → LLM-роутер для непонятного |
| Стек сервера | Python / FastAPI / Postgres |
| Приватность | Self-host, pluggable LLM (Ollama / API с минимумом данных) |
| Доставка пушей | Батч (не real-time) на старте |

## Порядок реализации (роутинг моделей)

| Фаза | Что | Модель |
|---|---|---|
| 0 | OpenAPI-контракт + схема БД + дизайн пайплайна | **opus** |
| 1 | Android: NotificationListener → очередь → ingest | sonnet |
| 2a | Сервер: ingestion + CRUD + grouping | sonnet |
| 2b | Сервер: rules-движок + LLM-роутер | opus |
| 3 | Сайт-дашборд «важное» | sonnet |

## Контракт v1 (см. `contracts/openapi.yaml`)

```
POST   /v1/devices:register
POST   /v1/notifications:ingest      (идемпотентно по client_id)
GET    /v1/items   ?importance&area&project&tag&status&from
GET    /v1/items/{id}
PATCH  /v1/items/{id}                done / snooze / reassign / dismiss
GET    /v1/groups
CRUD   /v1/areas /v1/projects /v1/rules
GET    /v1/tags
```

Авторизация: bearer-токен устройства. Идемпотентность ingest по `client_id`.
