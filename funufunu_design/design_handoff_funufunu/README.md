# Handoff: funufunu — морской редизайн Aggregat

Редизайн веб-дашборда агрегатора уведомлений (`web/` в репозитории `poledustaren/aggreagator`).
Метафора: **важность сообщения = погода на море** (Штиль → Рябь → Волны → Шторм → Ураган).
Название приложения: **funufunu** (заменяет «Aggregat» везде в UI).

## О файлах дизайна

Файлы в `design/` — **дизайн-референсы в HTML** (прототипы, показывающие внешний вид и поведение),
а НЕ production-код для копирования. Задача разработчика — **воссоздать эти макеты в существующем
стеке проекта**: React + TypeScript + Tailwind CSS (`web/src/`), с react-router и @tanstack/react-query,
используя уже существующие типы из `web/src/types/api.ts` (Item, Area, Project, Group, Process, ThemeNode).

- `design/funufunu Морская сводка.dc.html` — весь прототип: 5 экранов × (телефон + десктоп) × (тёмная + светлая тема). Открывается в браузере; данные и разметка — внутри файла.
- `design/StormCard.dc.html` — переиспользуемая карточка сообщения («шторм-карточка»).

## Fidelity

**High-fidelity (hifi).** Цвета, типографика, отступы, радиусы и состояния — финальные.
Воспроизводить пиксельно, но средствами существующего стека (Tailwind-темой, а не инлайн-стилями).

## Ключевой компонент: StormCard (замена ItemCard + ImportanceBadge)

Структура (см. `design/StormCard.dc.html`):
1. Полоса важности слева (4px, цвет шкалы).
2. Оверлайн: ПОГОДА (mono, цвет шкалы, uppercase) · время («2 мин»).
3. Заголовок + summary.
4. Справа: балл (21px mono, цвет шкалы) + **волна-спарклайн** — SVG 46×16, синусоида:
   `amp = 1.2 + (imp/100)·(h/2−1.6)`, `cycles = 1 + imp/22`, амплитуда нарастает слева направо (×(0.45+0.55·x/w)). stroke 1.6, цвет шкалы, opacity .9.
5. Чипы: источник (surface2), зона (цвет зоны + подложка 14% альфы), «→ suggested_action» (accent).
6. Действия: **Готово** (заливка accent, текст `#07141c`), **Отложить** (surface2), **Скрыть** (прозрачная). PATCH `/items/{id}` → status done/snoozed/dismissed. Свайпы из текущего ItemCard сохранить (вправо done, влево dismissed).

## Экраны

### 1. Сводка (`/`)
- **Герой «Морская сводка»**: тёмный градиент неба `linear-gradient(162deg,#0c3c4d,#0a1d29 72%)` в обеих темах; дрейфующие облака (2 размытых radial-gradient, `translateX` 15/19s alternate) и пунктирная «рябь» (repeating-linear-gradient, сдвиг фона 6s). Слева заголовок-прогноз («Штормовое предупреждение» при пике ≥81, «Свежий ветер, местами шторм» при ≥61, иначе «Спокойное море»). Справа **Балл шторма** = округлённое среднее importance инбокса + название градации. Внизу мини-гистограмма из 5 стихий (высота = count/max).
- Сегмент «Темы / Зоны» (как текущие режимы DigestPage): темы — раскрываемые карточки (полоса цвета максимума, счётчик, пилюля погоды, макс. балл, caret ▸/▾); зоны — группы с mono-заголовком цвета зоны.

### 2. Лента (`/feed`)
Фильтры-пилюли по стихиям (Всё / Ураган / Шторм / Волны / Спокойно=0–40) со счётчиками; активная — цвет стихии + подложка 16%. Ниже — StormCard-лента (сортировка по новизне).

### 3. Треды (`/groups`)
Карточка треда: иконка «течение» (кольцо+точка в цвете погоды), название, `источник · N сообщ. · время`, макс. балл. Раскрытие — вложенные StormCard + «+ ещё N в этом треде».

### 4. Процессы (`/processes`, `/timeline`)
Панель «Погодные фронты · 14 дней»: горизонтальные полосы-ганты (высота 14px, скругление 7px, градиент цвета важности → 50% альфы), позиция = (start/13, (end−start)/13) от ширины оси; ось — 6 mono-тиков. Ниже — список процессов (полоса 6×40px, заголовок, summary, балл, счётчик). Клик — выбор процесса (на десктопе деталь в правой панели, рамка выбранной карточки — цвет важности).

### 5. Связи (`/relations`)
SVG-граф (viewBox 640×400): узлы-циклоны — круг радиуса `14 + item_count·2`, заливка цветом важности, внутри балл (mono, белый), подпись снизу, glow-подложка (цвет, 16% альфы, r+11); выбранный — белая обводка 3px. Рёбра — линии по типам связей (см. токены). Легенда под графом.

## Каркас / навигация

- **Мобильный** (примерно ≤768px): контент одной колонкой (горизонтальные паддинги 16px), заголовок экрана сверху (оверлайн-дата mono + 25px Bricolage), **нижний таббар** (bg2, без верхней линии): 5 табов — иконка 22px + подпись 10px; активный = accent, неактивный = ink3.
- **Десктоп**: левый сайдрейл 212px (bg2): логотип, 5 пунктов (иконка 17px + текст 13px; активный — подложка `accent 16%` + текст accent), внизу счётчик «Разобрано сегодня». Контент max-width 720px. Правая панель 302px (bg2): «Обстановка» (5 строк: счётчик + прогресс-бар + название стихии), «Пиковые угрозы» (топ-3 по importance), деталь выбранного процесса.
- Иконки — line-SVG (viewBox 20×20, stroke 1.3–2.4, круглые концы): Сводка = компас (круг+стрелка), Лента = 3 волны, Треды = кольцо+точка, Процессы = 3 полосы разной длины, Связи = 3 узла+рёбра. Готовые path — в файле прототипа (блок таббара).

## Состояние и данные

- `screen` (5 вкладок), `theme` (dark/light, персистить в localStorage), `mode` темы/зоны, `feedSev` фильтр, `expanded` (раскрытые темы/треды), `selProc` (выбранный процесс), оптимистичные PATCH-мутации статусов (уже есть в `usePatchItem`).
- Балл шторма и распределение — считать из инбокса на клиенте (как ImportanceBuckets в `/stats/overview`, но 5 корзин по 20).
- Анимации: только герой (облака/рябь) и transition 0.2–0.4s на смену состояний; уважать `prefers-reduced-motion`.
